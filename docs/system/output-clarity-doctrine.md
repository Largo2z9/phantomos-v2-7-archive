1. *"L'opérateur peut-il scanner l'output en 5 secondes ?"* (density · whitespace · headers sobres)
2. *"Comprend-il sans glossaire ni doc tierce ?"* (dejargonisation · vocabulaire opérateur-facing)

4. *"L'output a-t-il la cohérence visuelle d'un produit shipped ?"* (iconographie unique · séparateurs cohérents · zéro emoji décoratif)

**Différenciation canon vs output alpha interne** ·

| Layer | Output alpha interne | PhantomOS canon Output Clarity |
|---|---|---|
| Iconographie | mixte (emoji couleur + symboles texte + decorative) | unique 5 symboles (✓ ◐ ○ ✗ ⚠) |

Output alpha interne signale qu'un produit n'est pas terminé. Output PhantomOS canon force la grammaire shipped product. C'est la différence entre démo R&D (passive) et produit livrable (operable).

## 3. Standards de référence canon · 6 principes shipped Vercel / GitHub-grade

Output canon respecte 6 principes shipped product · chaque principe est canon obligatoire (pas optionnel) ·

### Principe 1 · Un H1 · sections sobres · séparateurs prévisibles

Un seul titre principal par output. Sections H2 numérotées ou nommées sobrement. Séparateurs canon ━━━ entre macro-sections · ─── entre sous-sections. Structure prévisible cross-outputs · l'opérateur navigue par familiarité visuelle.

### Principe 2 · Un système iconographique constant · zéro emoji couleur

5 symboles canon uniques (✓ ◐ ○ ✗ ⚠) sur tous les outputs PhantomOS opérateur-facing. Zéro emoji couleur (🔥 🟢 🟡 🔴 🟠 🟣 ⚪️ ⚫). Zéro émoticône décoratif (📋 💡 ⚠️ 🧠 📥 ✅ 🚀). Cohérence visuelle = signal shipped product.

### Principe 3 · One thing per line · jamais empiler

Une métrique par ligne. Une information par ligne. Jamais empiler 3 facts disjoints sur même ligne (anti-pattern *"aujourd'hui · 0 test live · 5 audiences en hypothèse"*). Si 3 facts existent · 3 lignes ou tableau.

### Principe 4 · Termes courts opérateur-facing · zéro jargon interne

Vocabulaire opérateur-facing canon mapping `operator-vocabulary-translation.md`. Zéro jargon Phantom-interne dans output user-visible. Termes courts · directs · accessibles novice complet. Si terme technique nécessaire · encadrer backtick code + signal explicite.

### Principe 5 · Action items isolés du diagnostic · syntaxe stable

Section diagnostic et section actions strictement séparées. Action items format stable canon · *"N. Action verbalisée    rationale 1 ligne"*. Max 3 actions par output. Pas de mélange backtick + texte naturel + parenthèse rationale.

### Principe 6 · Density modérée · whitespace généreux

Rythme cohérent cross-output. Whitespace généreux entre blocs. 1 ligne vide entre paragraphes. Pas de bloc dense suivi de bloc sparse. Density = signal calibration shipped product · profil cognitif visuel scan 5 secondes.

## 4. Iconographie canon · système unique 5 symboles

Système iconographique canon strict v2.79.2+ ·

| Symbole | Sens canon | Usage |
|---|---|---|
| ✓ | complet · OK | état terminé · validation passée · check passed |
| ◐ | partiel | encoding partiel · fiche partielle · WIP |
| ○ | vide | aucune donnée · jamais encodé · slot blanc |
| ✗ | absent · non scopé | hors scope · non applicable · refusé |
| ⚠ | critique | alerte · drift · action immédiate requise |

**Règles canon iconographie** ·

- Légende symboles au pied de chaque output opérateur-facing (pas dans le titre · pas en haut)
- Ordre canon légende · `✓ complet  ◐ partiel  ○ vide  ✗ absent  ⚠ critique`
- BANNIR emoji couleur · 🔥 🟢 🟡 🔴 🟠 🟣 ⚪️ ⚫ · drapeau circulaire · drapeau carré · etc.
- BANNIR émoticônes décoratifs · 📋 💡 ⚠️ 🧠 📥 ✅ 🚀 🎯 📊 🔍 ⏰ 🛠 · etc.
- BANNIR emoji nationalité · 🇫🇷 🇺🇸 etc.
- BANNIR emoji status custom · 🟢 ACTIF / 🟡 PAUSE / 🔴 STALE remplacé par symboles canon

**Exception canon** · contextes hors-output (CHANGELOG.md · commits git · doc R&D interne) peuvent utiliser emoji décoratif. Output opérateur-facing user-visible (`/phantom` · `/bird` · `/breakdown` · skills synthesis stratégique · brief-day output · etc.) respecte canon strict.

## 8. Action items canon · format stable

Action items canon v2.79.2+ format stable strict ·

**Format canon** ·

```
N. Action verbalisée    rationale 1 ligne
```

**Règles canon action items** ·

- N max 3 actions par output (cap strict)
- Action verbalisée · verbe d'action direct · pas backtick command technique
- Rationale 1 ligne max · 60 chars cap
- Espacement canon · 4 spaces entre action et rationale (cohérent table alignement)
- Pas de mélange backtick + texte naturel + parenthèse rationale
- Skill names backstage · verbalize en action opérateur-facing
- Section ACTIONS isolée du diagnostic · jamais inline dans prose

**Exemple canon** ·

```
ACTIONS

1. Compléter fiche produit principal    données manquantes 4 champs core
2. Lancer cartographie audience         pivot strategy v2 nécessaire
3. Programmer review hebdo              cycle pacing canon hebdo
```

**Anti-pattern strict** ·

```
ACTIONS PRIORITAIRES

- Lance `produce-paid-angles --focus=audience-X` (parce que matrice ✓✓ PRIMARY)
- `mine-voc` sur cohort Y (verbatims sparse · seuil 5 verbatims canon)
- Run brief-day demain (background actif)
```

(jargon · backtick · parenthèse · mix syntaxe = invalid)

## 10. Redondance zero · une info · un format

Redondance canon v2.79.2+ strict ·

**Règle canon** · une info dite une seule fois · un seul format · un seul endroit.

**Détection redondance** ·

- Même fact apparaît 2+ fois cross-sections → refactor → 1 fois au bon endroit
- Métrique répétée header + body + footer → choisir 1 surface canon
- Stratégie résumée 3 fois (intro + body + close) → 1 fois en STRATÉGIE
- Audience listée 2 fois (matrice + body) → matrice seule · body référence

**Exemple anti-canon** ·

```
Output mentionne "0 test live" 3 fois ·
- Header EN COURS · "0 test live aujourd'hui"
- Body diagnostic · "Tu n'as lancé aucun test live cette semaine"
- Close · "Pense à lancer 1 test live"
```

**Pattern canon refactor** · 1 ligne ALERTES · *"0 test live ⚠ semaine"* · 1 action ACTIONS · *"Lancer 1 test live · cycle test canon hebdo"*.
