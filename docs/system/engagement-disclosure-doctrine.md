## 1. Thèse fondatrice

> Tout skill orchestrateur ou producer heavy DOIT exposer un disclosure pré-engagement · plan · ETA · démarche · demande confirmation explicite. L'opérateur s'engage en conscience claire de ce dans quoi il rentre. Pas de surprise. Pas de mi-parcours frustration. Pas de captivité silencieuse.

**Définition canon engagement disclosure** · ensemble des pratiques opérationnelles qui forcent l'agent à présenter un disclosure pré-engagement structuré (4 éléments obligatoires) AVANT le lancement de tout skill orchestrateur ou producer heavy. Disclosure canon répond à 4 questions miroir opérateur ·

1. *"Dans quoi je m'engage ?"* (Plan · N étapes ordonnées headers courts)
2. *"Combien de temps ça prend ?"* (ETA · estimation realistic minutes ou range)
3. *"Qu'est-ce qui se passe pendant ?"* (Démarche · ce que je vois · ce que je dois fournir · ce que j'obtiens)
4. *"Je valide ou je reporte ?"* (Confirmation explicite · close binaire)

**Différenciation canon vs LLM commodity** ·

| Layer | LLM commodity (ChatGPT · Claude · Notion AI) | PhantomOS canon engagement disclosure |
|---|---|---|
| Lancement skill lourd | silent post-trigger (opérateur captif) | disclosure pré-engagement obligatoire |
| Plan exposé | absent (boîte noire opaque) | canon (N étapes ordonnées max 8) |
| ETA chiffré | vague ou absent (*"un instant"* · *"patience"*) | canon (minutes ou range realistic) |
| Démarche explicit | absente (opérateur subit le cycle) | canon (ce qu'il voit · fournit · obtient) |
| Confirmation | implicite (lance par défaut) | canon (close binaire OK / J'attends) |
| Opt-out expert | absent | canon (memory `disclosure_preference: silent` post-N fois) |

ChatGPT lance silencieusement ses chains opaques · l'opérateur découvre le runtime au fur et à mesure. PhantomOS canon expose le contrat pré-engagement · l'opérateur valide consciemment OU reporte. C'est la différence entre un service magique (passive) et un atelier transparent (operable).

Cette doctrine canonise le pattern systémique cross-skills orchestrators + producers heavy. Tout skill avec `type: orchestrator` OR runtime expected >5 min OR spans sessions OR 2+ sub-skills déployés OR producer heavy paid livrable client-facing respecte le pattern disclosure pré-engagement.

## 2. Le problème résolu

Sans Engagement Disclosure Doctrine canon ·

1. **Captivité silencieuse opérateur.** Skills orchestrateurs lourds (build-atlas-complete 25-40 min · onboard-brand 20-30 min · creative-brief-composer 10-15 min · produce-paid-matrix 15-25 min · mine-voc 5-15 min · audit-meta-account 10-20 min) lancent silencieusement post-trigger opérateur. Opérateur découvre durée mi-parcours · frustration *"j'aurais préféré savoir avant"*. Adoption friction permanent.

2. **Asymétrie information agent vs opérateur.** Agent sait (skill orchestrateur lourd prend 30 min · déploie 4 sub-skills · spans 2 sessions). Opérateur croyait (instantané ou rapide quelques minutes max). Décalage permanent · trust cassé · sentiment manipulation post-fait.

3. **Pas de filet de sécurité validation.** Opérateur trigger skill par réflexe sans conscience profondeur engagement. 25 min plus tard · opérateur réalise qu'il aurait préféré chain alternative OR différer OR fournir input plus structuré upfront. Coût rework évitable.

4. **Cycle multi-sessions opaque.** Skills spans sessions (onboard-brand semaines 1-4 · build-atlas-complete continuous enrichment) lancent sans annonce durée multi-sessions. Opérateur croit one-shot · découvre semaines de cycle. Désengagement silent · workspace abandonné.

5. **Sub-agents parallèles invisible.** Agent main délégue 2+ sub-skills via Task tool silent (snapshot-brand + mine-voc + define-specs parallèle). Opérateur découvre travail sub-agents post-fait · pas de conscience profondeur orchestration. Pattern miroir LLM black box.

6. **Producer heavy livrable client-facing sans validation.** Skills paid (produce-paid-angles · produce-copy-brief · compose-creative) produisent livrable client-facing 15-30 min runtime sans demander confirmation upfront *"OK pour lancer ?"*. Opérateur récupère output non-aligné · rework permanent.

Engagement Disclosure Doctrine = doctrine canon qui ferme ces 6 gaps via 5 triggers canon obligatoires + 4 éléments disclosure pattern + 8 Hard Rules enforcement runtime + opt-out expert explicit.

## 3. Triggers canon obligatoires · 6 conditions qui forcent le disclosure pré-engagement

Disclosure pré-engagement OBLIGATOIRE dès qu'un des 6 triggers canon se déclenche · chaque trigger est canon (pas optionnel) ·

### Trigger 1 · Type orchestrator

Skill avec `type: orchestrator` dans frontmatter SKILL.md. Skills canon · `build-atlas-complete` · `onboard-brand` · `creative-brief-composer` · `produce-paid-matrix` · `launch-intelligence-deep`. Routing automatique · agent lit frontmatter `type` avant exécution · si `orchestrator` détecté · disclosure pré-engagement obligatoire.

### Trigger 2 · Duration estimée >5 min

Runtime expected >5 min même pour skill non-orchestrator. Skills canon · `mine-voc` 5-15 min · `audit-meta-account` 10-20 min · `decompose-ad` 5-10 min · `snapshot-brand` 8-12 min (full 5 mouvements). Routing automatique · agent lit `expected_duration_minutes` frontmatter ou estimate basé sur skill type · si >5 min · disclosure pré-engagement obligatoire.

### Trigger 3 · Spans sessions

Cycle multi-sessions (skill produit livrable étalé sur 2+ sessions). Skills canon · `onboard-brand` semaines 1-4 · `build-atlas-complete` continuous enrichment · `produce-paid-matrix` build initial puis enrichment cycle. Routing automatique · agent lit `spans_sessions: true` frontmatter · si vrai · disclosure pré-engagement obligatoire avec annonce durée multi-sessions explicit.

### Trigger 4 · 2+ sub-skills déployés

Agent main délégue 2+ sub-skills via Task tool (orchestration multi-agent). Pattern canon · `build-atlas-complete` déploie snapshot-brand + mine-voc + define-specs + profile-audience + produce-paid-angles parallèle. Routing automatique · agent calcule pre-execution nombre sub-skills planifiés · si ≥2 · disclosure pré-engagement obligatoire avec annonce orchestration multi-agent explicit.

### Trigger 5 · Producer heavy paid

Skill producer paid heavy avec output livrable client-facing critique. Skills canon · `produce-paid-angles` (formula Obs+Tension+Reframe+Bridge) · `produce-copy-brief` (écrit court ou long persuasion) · `compose-creative` (composition pub finale). Routing automatique · agent lit `output_classification: livrable_client_facing` frontmatter · si vrai · disclosure pré-engagement obligatoire avec annonce critique business explicit.

### Trigger 6 · Intelligence compositionnelle (atlas × métier)

Skills qui raisonnent sur brand encoded (atlas substrat) × domaine métier (audience · pain · canal · query · positionnement · paid · creative). Composent intelligence émergente cross-cell atlas. Skills canon · `mine-voc` · `mine-vom` · `profile-audience` · `produce-paid-angles` · `produce-paid-matrix` · `produce-copy-brief` · `compose-creative` · `build-atlas-complete` · `creative-brief-composer` · `decompose-ad` · `adapt-from-competitor` · `audit-meta-account` · `analyze-perf` · `routine-perf` · `score-matrix` · `snapshot-brand` (post-IP synthesis stratégique). Obligation NEW v2.79.5 · NIVEAU 0 paramètres décomposés explicit pre-exec (audience cible · pains prioritaires · canaux · queries · hypothèses figées · biais à éviter avec POURQUOI pour chaque). Routing automatique · agent détecte skill scope intelligence compositionnelle pre-execution · disclosure enrichi obligatoire (pattern v2.79.5+ Plan + Paramètres décomposés + ETA + Démarche + Confirmation). Cross-ref `decomposition-visibility-doctrine.md` v2.79.5+ NIVEAU 0.

**Pattern canon multiplicatif.** Les 6 triggers sont OR · l'activation d'un seul trigger suffit pour déclencher le disclosure. Trigger 6 enrichit pattern disclosure (5 éléments au lieu de 4 · paramètres décomposés ajouté). Pattern miroir équation système opérationnel (`operational-system-doctrine.md`) (déclenchement canon enforcement runtime).

## 4. Pattern disclosure canon · 5 éléments obligatoires (v2.79.5)

Disclosure pré-engagement canon décompose en 5 éléments orthogonaux mesurables et présentables · pattern v2.79.5 enrichi avec NEW élément Paramètres décomposés obligatoire pour skills intelligence compositionnelle (Trigger 6) ·

```
ÉLÉMENT 1 · Plan (N étapes ordonnées · max 6-8)
   ↓
ÉLÉMENT 2 · Paramètres décomposés (NEW v2.79.5 · skills intelligence compositionnelle Trigger 6)
   ↓
ÉLÉMENT 3 · ETA (chiffré minutes ou range realistic)
   ↓
ÉLÉMENT 4 · Démarche (implication + livrable)
   ↓
ÉLÉMENT 5 · Confirmation (close binaire explicit)
```

**Pattern canon multiplicatif.** Les 5 éléments sont multiplicatifs sur skills intelligence compositionnelle (Trigger 6) · l'absence d'un seul élément fait s'effondrer la qualité disclosure systémique. Pattern miroir équation système opérationnel (`operational-system-doctrine.md`) (multiplicatif strict). **Backward compat strict additif** · skills mécaniques type validate · capture · scaffold (Triggers 1-5 uniquement · pas Trigger 6) conservent pattern v2.79.3 (4 éléments · Plan + ETA + Démarche + Confirmation). Paramètres décomposés obligatoire UNIQUEMENT sur Trigger 6 activé.

### Élément 1 · Plan

N étapes ordonnées avec headers courts (max 6-8 étapes cap strict). Verbalisation opérateur-facing canon (zéro jargon interne · zéro skill names exposés · zéro field paths). Format `1. {Étape verbalisée}` numéroté · 1 ligne par étape. Headers courts FR sobres cohérent `output-clarity-doctrine.md` v2.79.2 HR-OCD-3.

### Élément 2 · Paramètres décomposés (NEW v2.79.5 · skills intelligence compositionnelle uniquement)

6 éléments canon obligatoires décomposés AVANT exécution skill intelligence compositionnelle (atlas × métier · Trigger 6). Reverse engineering visible pre-exec · l'opérateur valide consciemment paramètres avant lancement ·

- **Audience cible** · segment atlas brand consommé (audience_id · slug) · POURQUOI ce segment (pas un autre · stage filter · priorité)
- **Pains prioritaires** · liste ordonnée pains depuis atlas audience · POURQUOI cet ordre (densité verbatims · pain dominant)
- **Canaux / sources** · ranking sources mining (Reddit · Trustpilot · YouTube · etc.) · POURQUOI ce ranking (vernaculaire · densité)
- **Queries / critères opératoires** · mots-clés primaires + variants vernaculaires + critères tri · POURQUOI dimensions (couverture pain · matching audience)
- **Hypothèses figées** · ce que l'agent présume SANS validation · explicit OPÉRATEUR PEUT CHALLENGER avant lancement
- **Biais à éviter** · biais structurels corpus/source (Reddit skew · Trustpilot rage · YouTube paid · cherry-pick) · explicit risk flag pre-exec

Format canon · table 6 lignes avec POURQUOI colonne. Pattern miroir `decomposition-visibility-doctrine.md` v2.79.5+ NIVEAU 0. Trigger 6 active obligatoirement · skills mécaniques (Triggers 1-5 uniquement) skip cet élément.

### Élément 3 · ETA

Estimation realistic chiffrée minutes ou range. Format canon `~{X-Y minutes}` OR `~{X minutes}` cap strict. JAMAIS *"rapide"* · *"long"* · *"un instant"* · *"patience"* · *"vite"*. Realistic = lower bound observée best case · upper bound observée worst case. Range si variance >30% · single chiffre si variance <30%.

### Élément 4 · Démarche

Ce qui se passe dans le cycle · décomposé 2 facettes ·
- **Implication opérateur** · ce qu'il doit fournir pendant le cycle (input · validation · décisions · accès credentials). Format 1 ligne sobre.
- **Livrable final** · ce qu'il obtient à la fin (output verbalisé opérateur-facing · pas plumbing). Format 1 ligne sobre.

### Élément 5 · Confirmation

Close binaire explicit obligatoire. Format canon `OK avec ces paramètres ? Tu ajustes lequel avant que je lance ?` (variante intelligence compositionnelle Trigger 6) · `OK pour lancer ? · ou tu préfères attendre / faire autre chose` (variante skills mécaniques Triggers 1-5). Variantes acceptables · `Lance / Plus tard` · `OK / J'attends`. JAMAIS opt-in default silent (agent lance sans validation). JAMAIS menu 4+ options equal-weight. Toujours close binaire explicit.

## 7. Anti-pattern court-circuit

Agent NE PEUT PAS skip disclosure même si opérateur a lancé skill avec syntax explicite (e.g. `/build-atlas-complete stepprs` direct). Trigger lancement skill par opérateur ≠ validation disclosure pré-engagement. Pattern canon · agent reçoit trigger → calcule si un des 5 triggers canon s'active → si oui · disclosure obligatoire AVANT exécution → si non · exécution direct.

**Court-circuit autorisé UNIQUEMENT 2 conditions explicit** ·

1. **Flag `--no-disclosure` explicit** · opérateur trigger skill avec flag explicit *"je sais ce que je lance · skip disclosure"*. Pattern canon · agent skip disclosure UNIQUEMENT sur flag explicit syntax · jamais inférence.

2. **Memory operator opt-out** · `operator/profile.json#preferences.disclosure_preference: silent` set par opérateur EXPLICIT post-N fois (mécanisme canon opt-out expert · cf §10).

JAMAIS court-circuit silent (skip disclosure sans flag explicit ni memory opt-out). Violation = bug invalid pattern canon · trust cassé opérateur.

## 10. Memory operator opt-out

Opérateur peut set `operator/profile.json#preferences.disclosure_preference: silent` post-N fois pour skip disclosure cross-skills orchestrators. Mécanisme canon opt-out expert · jamais opt-in default.

**Pattern canon opt-out explicit** ·

1. **Phase 1 · Default canon** · disclosure pré-engagement obligatoire premières N invocations skills orchestrators (N = 3-5 sessions typique)
2. **Phase 2 · Détection pattern** · agent détecte opérateur validate disclosure systematiquement OK sans drill (pattern *"je connais le cycle"*)
3. **Phase 3 · Suggestion opt-out** · agent surface suggestion *"tu veux passer en mode silent pour les skills orchestrators ? · tu connais le cycle · économie 30 secondes par invocation"*
4. **Phase 4 · Activation explicit** · opérateur confirme explicit → agent set `disclosure_preference: silent` dans `operator/profile.json`
5. **Phase 5 · Skip canon** · disclosure pré-engagement skip post-set memory · agent lance skill direct

**Pattern canon réversible** · opérateur peut revert `disclosure_preference: verbose` à tout moment via *"remets-moi les disclosures · je préfère"*. Memory append-only event log via `write_to_context()` mutation gate.

**Anti-pattern strict** · agent set `disclosure_preference: silent` sans confirmation explicit opérateur (opt-in default disguisé). Violation = bug invalid pattern canon.

## 11. Hard Rules canon (HR-EDD-1 à HR-EDD-8)

### HR-EDD-1 · 5 triggers canon obligatoires déclenchent disclosure

Disclosure pré-engagement obligatoire dès qu'un des 5 triggers canon s'active (orchestrator type · duration >5 min · spans sessions · 2+ sub-skills · producer heavy paid). Routing automatique agent pre-execution. Violation = bug invalid pattern canon (skill lance sans disclosure).

### HR-EDD-2 · Pattern disclosure 4 éléments obligatoires

Disclosure rendu respecte 4 éléments canon strict · Plan + ETA + Démarche + Confirmation. Multiplicatif · l'absence d'un seul élément fait s'effondrer la qualité disclosure. Violation = bug invalid pattern canon (disclosure incomplet).

### HR-EDD-3 · Confirmation explicite avant lancement · close binaire

Disclosure ferme sur close binaire explicit obligatoire (`OK / J'attends` · `Lance / Plus tard` · variantes acceptables). JAMAIS opt-in default silent (agent lance sans validation). JAMAIS menu 4+ options equal-weight. Violation = bug invalid pattern canon.

### HR-EDD-4 · Skills navigator/capturer/validator légers exclus <2 min

Disclosure NON-applicable skills légers <2 min runtime type navigator/capturer/validator. Routing canon agent lit `expected_duration_minutes` + `type` frontmatter pre-execution. Friction excessive disclosure pour skill quick = violation no_overengineer canon. Violation = bug invalid pattern canon (disclosure pour skill léger).

### HR-EDD-5 · Court-circuit autorisé UNIQUEMENT opérateur opt-out explicit

Agent skip disclosure UNIQUEMENT 2 conditions explicit · (1) flag `--no-disclosure` syntax explicit OR (2) memory `disclosure_preference: silent` set par opérateur EXPLICIT post-N fois. JAMAIS court-circuit silent. Violation = bug invalid pattern canon (skip disclosure sans opt-out explicit).

### HR-EDD-6 · ETA realistic chiffrée minutes ou range · jamais vague

ETA disclosure chiffré realistic format `~{X-Y minutes}` OR `~{X minutes}` cap strict. JAMAIS *"rapide"* · *"long"* · *"un instant"* · *"patience"* · *"vite"*. Range si variance >30% · single chiffre si variance <30%. Violation = bug invalid pattern canon (ETA vague).

### HR-EDD-7 · Plan N étapes ≤8 max · headers courts
