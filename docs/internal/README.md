# Internal Documentation

⚠️ **FOR CONTRIBUTORS AND CORE MAINTAINERS ONLY**

If you are using PhantomOS as an operator (a DTC media partner or workspace user), you don't need to read this section. See [`docs/product/`](../product/) for operator guidance instead.

This section documents the architecture, governance, and internal disciplines that govern how PhantomOS is built, extended, and maintained.

## What's here

- **`canon.md`** — the canonical reference for internal vocabularies and design decisions locked into this workspace.
- **`releases/`** — technical release manifests, schema migrations, breaking changes, and dependency updates for contributors integrating new versions.

## For contributors

Start with the architecture overview in `docs/system/architecture.md`, then consult `docs/system/extending.md` to understand the extension patterns.
