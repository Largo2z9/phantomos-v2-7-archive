# Vision docs

Public-facing narrative. Why PhantomOS exists, what it argues, what it ships next.

## Files

- **`prisms.md`** — six framings of PhantomOS, each for a specific audience. Canonical snippets used across the product surface.
- **`manifesto.md`** — the thesis. The macro shift, the gap in the market, the method, the risks. Sourced, opinionated, durable.
- **`roadmap.md`** — what ships next and why. Versioned.

## Who this is for

Anyone evaluating PhantomOS — investors, operators still deciding, consultants curious about the method, journalists, future contributors. Also the maintainer in six months, reading their own thesis to check it still holds.

## Conventions

Vision docs are the only place in the doc surface where name-dropping is allowed (in `manifesto.md`, sourced explicitly). Prisms and roadmap stay name-drop-free. Opinionated, narrative, durable over clever.
